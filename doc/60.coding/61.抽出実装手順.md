# 実装手順書：Salesforceデータ抽出・S3保存バッチ (AppFlow)

## 1. Salesforce 側の準備

AppFlowがSalesforceに接続するための「入り口」を作成します。

1. **接続アプリケーション (Connected App) の作成**
* [設定] > [アプリケーションマネージャー] > [新規接続アプリケーション] を選択。
* **接続アプリケーション名**: `AWS_AppFlow_Connector`（任意）
* **API（OAuth設定の有効化）**: チェックを入れる。
* **コールバック URL**: `https://api.amazon.com/auth/o2/token/success`（AWSリージョンに関わらず共通）
* **選定された OAuth 範囲**:
* `APIを使用してユーザーデータを管理 (api)`
* `いつでもリクエストを実行 (refresh_token, offline_access)`


* **保存**: 表示される `コンシューマ鍵 (Client ID)` と `コンシューマの秘密 (Client Secret)` をメモします。


2. **統合ユーザーの確認**
* 連携専用のユーザーに「APIの有効化」および対象オブジェクトへの「参照権限」があることを確認します。



---

## 2. Amazon S3 側の準備

抽出データの保存先となるバケットを用意します。

1. **S3 バケットの作成**
* 名前: `sf-data-extract-bucket-[AWSアカウントID]`
* リージョン: AppFlowを実行するリージョンと同じにする。


2. **フォルダ構造の作成**（空のフォルダを作成するか、AppFlow実行時に自動生成させる）
* `extract/Account/`



---

## 3. Amazon AppFlow の接続設定 (Connector Profile)

AWSコンソールでSalesforceへの接続パスを作成します。

1. **AppFlow コンソール**を開き、[接続] > [Salesforce] を選択。
2. **[接続を作成]** をクリック。
* **OAuth 2.0 設定**:
* `コンシューマ鍵` と `コンシューマの秘密` を入力。
* インスタンスURL: 通常は `https://login.salesforce.com`。


* **接続名**: `Salesforce_Production_Conn`


3. **認証の実行**: ポップアップが表示されるので、Salesforceの統合ユーザーでログインし「許可」をクリック。
* これにより、Secrets Managerに認証情報が自動保存されます。



---

## 4. フロー (Flow) の作成と詳細設定

1. **フローの作成**
* [フロー] > [フローを作成]
* **フロー名**: `Extract_Salesforce_Account_Daily`


2. **データソースの設定**
* **送信元システム**: `Salesforce`
* **Salesforce 接続**: 手順3で作成した接続を選択。
* **Salesforce オブジェクト**: `Account` (取引先)
* **API 設定**: 「Salesforce Bulk API 2.0」にチェックを入れる。


3. **送信先の設定**
* **送信先システム**: `Amazon S3`
* **バケットの詳細**: 手順2で作成したバケットを選択。
* **ファイル形式設定**: `CSV`
* **ファイル集約**: `ファイルを結合する`（ファイル数を抑えるため推奨）。


4. **マッピングと変換**
* **マッピング方法**: `すべてのフィールドを直接マッピングする` を選択（必要に応じて不要な項目を削除）。
* **検証 (Validation)**: データ不備時のアクション（例: レコードのスキップ）を任意で設定。


5. **実行タイミング (Trigger)**
* **フロー実行のスケジュール設定**:
* `スケジュールどおりにフローを実行` を選択。
* **転送モード**: `増分転送`。
* **判定フィールド**: `LastModifiedDate` (最終更新日時)。
* **頻度**: 毎日/毎時など要件に合わせて設定。





---

## 5. 監視・通知の実装 (EventBridge)

実行失敗を検知する仕組みを追加します。

1. **EventBridge ルールの作成**
* 名前: `AppFlow_Failure_Notification`
* イベントパターン:
```json
{
  "source": ["aws.appflow"],
  "detail-type": ["AppFlow Flow Execution Status Change"],
  "detail": {
    "status": ["Execution Failed"]
  }
}

```




2. **ターゲット設定**
* **SNS トピック**: あらかじめ作成しておいた通知用SNSトピックを指定し、メールアドレスなどをサブスクライブさせます。



---

## 6. 実装完了後のテスト

1. **手動実行**: 画面右上の [フローを開始] をクリックし、ステータスが `Success` になるか確認。
2. **S3 確認**: S3バケットにCSVファイルが作成され、中身が正しく抽出されているか確認。
3. **増分テスト**: Salesforce側で1レコードだけ更新し、次回のスケジュール実行でその1件だけが抽出されるか確認。
