Salesforceにおいて、AWS AppFlowなどの外部システムと連携するための**統合専用ユーザー（Integration User）**を作成する手順をまとめました。

セキュリティとコストの観点から、Salesforceが提供している専用の「Salesforce Integration」ライセンスを利用するのがベストプラクティスです。

---

# Salesforce 統合専用ユーザー作成手順書

## 1. 権限セット（Permission Set）の作成

ユーザーに直接権限を付けるのではなく、権限セットを作成して割り当てることで、管理が容易になります。

1. **[設定]** > **[ユーザー]** > **[権限セット]** を開きます。
2. **[新規]** をクリックします。
* **表示名**: `AWS_AppFlow_Integration_Permissions`(Read、Read/Write権限を分けるか)
* **ライセンス**: `--なし--` または `Salesforce Integration`（ライセンスを固定する場合は後者）


3. **[保存]** をクリックします。
4. 権限を編集します：
* **システム権限**: **「API の有効化」** にチェックを入れます。
* **オブジェクト設定**: 連携対象（Account, Opportunity等）の「参照」「作成」「編集」および各項目の参照・編集権限を付与します。



---

## 2. 統合専用ユーザーの新規作成

実際に連携に使用するアカウントを作成します。

1. **[設定]** > **[ユーザー]** > **[ユーザー]** を開きます。
2. **[新規ユーザー]** をクリックします。
3. 以下の項目を入力します：
* **姓 / 名**: `AppFlow` / `Integration`
* **メールアドレス**: 管理者の共有アドレス（通知を受け取れるもの）
* **ユーザー名**: `aws.appflow.integration@company.com`（世界で一意である必要あり）
* **別名**: `appflow`
* **ユーザーライセンス**: **Salesforce Integration**
* **プロファイル**: **Salesforce API Only System Integrations**


4. **[保存]** をクリックします。設定したメールアドレスにパスワード設定メールが届きます。

---

## 3. 権限の割り当て

作成したユーザーに、手順1で作った権限セットを紐付けます。

1. 作成したユーザーの詳細画面を開きます。
2. **[権限セットの割り当て]** セクションで **[割り当ての編集]** をクリックします。
3. 手順1で作成した `AppFlow_Integration_Permissions` を「有効化された権限セット」へ移動し、**[保存]** します。

---

## 4. セキュリティ・運用設定（重要）

バッチ処理が途切れないよう、以下の設定を確認してください。

### 4.1 パスワードポリシーの変更

1. 作成したユーザーに紐づくプロファイル（または権限セット）の **[パスワードポリシー]** を確認します。
2. **「パスワードの有効期間」** を **「無期限」** に設定することを検討してください。
* *注意: OAuth (AppFlow) を利用する場合、リフレッシュトークンが有効であればパスワード期限が切れても接続は維持されますが、無期限設定の方が運用トラブルを回避できます。*



### 4.2 ログインIP範囲の制限（オプション）

* セキュリティをさらに高める場合、プロファイルレベルで **「ログイン IP 範囲」** を設定し、AWS AppFlow の IP アドレス帯域からのみアクセスを許可するように制限します。

---

## 5. 疎通確認

1. AWS AppFlow のコンソールに戻ります。
2. 作成したユーザー名とパスワードを使用して、接続（コネクタプロファイル）の認可が正常に完了することを確認します。
