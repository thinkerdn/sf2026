セキュリティと運用の安全性を最大限に高めるため、**「抽出用（読み取り専用）」**と**「取り込み用（読み書き用）」**でユーザーを分けて作成する場合の推奨設定値を提案します。

ユーザー名を分けることで、Salesforceのレコード更新履歴（最終更新者）を見た際に、「どのフロー（抽出 or 取込）に関連した動作か」がひと目で判別できるようになります。

---

## 1. 抽出用ユーザー（Salesforce → S3）の推奨設定値

このユーザーはデータの参照のみを行い、Salesforce側のデータを変更する権限を持ちません。

| 項目名 | 設定値の例 | 備考 |
| :--- | :--- | :--- |
| **姓 (Last Name)** | `AppFlow` | システム名を姓に設定します。 |
| **名 (First Name)** | `Extractor` | 役割（抽出）を名に設定します。 |
| **ユーザー名 (Username)** | `aws.appflow.extract@example.com` | 世界で一意のメール形式である必要があります。 |
| **別名 (Alias)** | `af_ext` | リストビュー等での識別用です。 |
| **ユーザーライセンス** | **Salesforce Integration** | 統合専用ライセンスを選択します。 |
| **プロファイル** | **Salesforce API Only System Integrations** | API限定アクセスを強制します。 |
| **割り当てる権限セット** | `AppFlow_ReadOnly_Access` | 前述の「参照のみ」の権限セットを付与します。 |

---

## 2. 取り込み用ユーザー（S3 → Salesforce）の推奨設定値

このユーザーはSalesforceへの書き込み（Upsert）を行うため、編集権限を持ちます。

| 項目名 | 設定値の例 | 備考 |
| :--- | :--- | :--- |
| **姓 (Last Name)** | `AppFlow` | 抽出用と揃えることでシステム連携であることを明示します。 |
| **名 (First Name)** | `Loader` | 役割（書き込み/ロード）を名に設定します。 |
| **ユーザー名 (Username)** | `aws.appflow.load@example.com` | 抽出用とは別のユニークな名前を設定します。 |
| **別名 (Alias)** | `af_load` | |
| **ユーザーライセンス** | **Salesforce Integration** | 同様に統合専用ライセンスを使用します。 |
| **プロファイル** | **Salesforce API Only System Integrations** | |
| **割り当てる権限セット** | `AppFlow_ReadWrite_Access` | 前述の「編集・作成」の権限セットを付与します。 |

---

## 3. この構成によるメリット



* **監査ログの明確化**: レコードの最終更新者が `Loader AppFlow` と表示されれば、S3からのデータ同期によって更新されたことが即座にわかります。
* **誤操作の防止**: 抽出設定（AppFlow側）で誤って「送信先をSalesforce」にしてしまっても、`Extractor` ユーザーには書き込み権限がないため、Salesforce側でエラーとなりデータ破壊を防げます。
* **パスワード管理**: 用途ごとに接続（コネクタプロファイル）を分けることで、一方の認証情報を変更しても、もう一方のフローに影響を与えずにメンテナンスが可能です。
