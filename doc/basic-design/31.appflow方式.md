Amazon AppFlowを使用して、Salesforceからデータを抽出し、Amazon S3にアップロード（保存）する機能

---

# 概要設計：Salesforceデータ抽出・S3保存バッチ (AppFlow版)

## 1. システム構成図

1. **Salesforce**: データソース（抽出元）
2. **AWS AppFlow**: データ転送エンジン（認証・抽出・変換・転送）
3. **AWS Secrets Manager**: Salesforce接続情報の管理（AppFlowが内部的に利用）
4. **Amazon S3**: データ保存先ストレージ（データレイク）
5. **Amazon CloudWatch**: 実行ログおよびエラー監視

---

## 2. 機能詳細設計

### 2.1 実行トリガー (Flow Trigger)

* **実行方式**: スケジュール実行（オンデマンド実行も可能）。
* **スケジュール設定**: 毎日/毎時など、ビジネス要件に合わせた頻度を設定。
* **増分転送（Incremental Transfer）**: 前回実行時からの差分（最終更新日時ベース）のみを抽出する設定を推奨。

### 2.2 データ抽出 (Source: Salesforce)

* **API種別**: **Salesforce Bulk API 2.0** を優先的に使用（大量データ抽出時のパフォーマンス向上とAPI制限の節約のため）。
* **抽出対象**: 任意のSalesforceオブジェクト（Account, Contact, Opportunity, カスタムオブジェクト等）。
* **フィルタリング**: 必要に応じて、特定の条件（例：Status = 'Closed'）に合致するレコードのみを抽出。

### 2.3 データ加工・マッピング (Task/Mapping)

* **フィールドマッピング**: Salesforceの項目名とS3に出力するカラム名を1:1でマッピング。
* **データ変換**: 必要に応じて、特定項目のマスキングや、形式の変換（タイムスタンプ形式の変更など）をAppFlow上で行う。

### 2.4 データ保存 (Destination: S3)

* **ファイル形式**: CSV, JSON, または Apache Parquet（分析用途に最適）。
* **パーティショニング**:
* `s3://[BucketName]/salesforce/[ObjectName]/YYYY/MM/DD/` 形式で保存。


* **ファイル集約**: 小さなファイルが大量に生成されるのを防ぐため、必要に応じてファイルの結合（Aggregation）設定を有効化。

---

## 3. ベストプラクティスに基づく設計ポイント

### ① セキュリティ (Security)

* **認証**: OAuth 2.0 認可コードフローを使用。認証情報は AWS Secrets Manager で保護。
* **暗号化**: S3に保存されるデータは、AWS KMS (SSE-KMS) を用いてサーバーサイドで暗号化。
* **最小権限**: AppFlowがS3バケットに書き込むためのIAMロールは、対象バケットのみに制限。

### ② 信頼性と監視 (Monitoring)

* **CloudWatch Logs**: フローの実行詳細ログを記録し、エラー発生時の調査を容易にする。
* **アラート通知**: EventBridge と SNS を連携させ、フローの失敗時に運用者へ即時通知（メール/Slackなど）。

### ③ スケーラビリティとコスト

* **API制限の考慮**: Salesforceの24時間APIリクエスト制限に抵触しないよう、一括処理（Bulk API）設定を活用。
* **ライフサイクル管理**: S3上の古いデータを一定期間後に Glacier へ移行、または自動削除するライフサイクルポリシーを設定。

---

## 4. 異常系処理

| 事象 | 対応方針 |
| --- | --- |
| **Salesforce メンテナンス** | スケジュール実行のリトライ設定、または次回の増分実行での補完。 |
| **S3 書き込みエラー** | IAM権限およびバケットポリシーの再確認。 |
| **認証トークンの失効** | AppFlowのコネクタ設定から再認可（再ログイン）を実施。 |
