# 概要設計：Salesforceデータ抽出・S3保存バッチ (Lambdaカスタム版)

## 1. システム構成図

---

## 2. 機能概要

EventBridge Schedulerをトリガーに、AWS LambdaがSalesforce APIを直接呼び出し、データを抽出してS3へ保存します。外部ライブラリ（`simple-salesforce` 等）を使用せず、Python標準の `urllib` 等を利用することで、脆弱性管理のコストを最小化します。

---

## 3. 詳細設計

### 3.1 実行制御 (EventBridge)

* **トリガー**: **EventBridge Scheduler**。
* **設定**: 定期的なスケジュール（例：毎日 01:00 JST）を `cron` で設定。
* **エラー制御**: 最大リトライ回数とデッドレターキュー（SQS）を設定。

### 3.2 認証処理 (JWT Bearer Flow)

セキュリティと自動実行の観点から **OAuth 2.0 JWT Bearer Flow** を採用します。

* **秘密鍵管理**: **AWS Secrets Manager**。
* **実装方式**:
* `json`, `base64`, `hashlib`, `hmac` などの標準ライブラリのみを使用。
* 署名処理（RS256）を自前で構成、または環境が提供するライブラリを最小限に利用。


* **フロー**:
1. Secrets Manager から秘密鍵と Client ID を取得。
2. 自前で署名済み JWT を生成。
3. Salesforce トークンエンドポイントへ POST し、アクセストークンを取得。



### 3.3 データ抽出ロジック (Bulk API 2.0)

Lambdaのタイムアウト（15分）を考慮し、大規模データにも対応可能な **Bulk API 2.0 (Query Job)** を使用します。

1. **ジョブ作成**: `POST /services/data/vXX.X/jobs/query`。
2. **待機**: ジョブ完了をポーリング（短時間で終わらない場合は Step Functions への移行を検討）。
3. **データ取得**: 完了後、CSV形式で結果をダウンロード。

### 3.4 データ保存 (S3)

* **パス構成**: `s3://[BucketName]/raw/salesforce/[ObjectName]/year=YYYY/month=MM/day=DD/`
* **形式**: CSV（Salesforce Bulk API のネイティブ形式）。
* **暗号化**: **SSE-KMS**。

---

## 4. 運用・セキュリティ設計

### 4.1 IAM 最小権限ポリシー

Lambdaに付与する権限を以下の3点に絞り込みます。

* **Secrets Manager**: 指定したシークレットの `GetSecretValue`。
* **S3**: 指定バケットへの `PutObject`。
* **CloudWatch Logs**: ロググループへの `CreateLogStream`, `PutLogEvents`。

### 4.2 監視設計

* **CloudWatch Alarms**:
* Lambdaの「エラー数」および「実行時間」を監視。
* 失敗時は **Amazon SNS** を介して通知。


* **ログ管理**: すべてのAPIリクエストのステータスコードと処理レコード数をログ出力。

---

## 5. 異常系処理

| 事象 | 検知方法 | 対応 |
| --- | --- | --- |
| **認証失敗 (401)** | Lambda ログ | 証明書の有効期限と Secrets Manager の値を確認。 |
| **API制限超過 (403)** | Lambda ログ | 実行間隔の調整、または抽出フィルタ（WHERE句）の強化。 |
| **タイムアウト** | CloudWatch | データ量が多い場合は、Step Functions でジョブ完了監視をループ化。 |

---

> **設計のポイント:**
> オープンソースを使用しないことで、ライブラリのアップデート作業が不要になり、長期的な保守性が向上します。一方で、JWTの署名（RS256）ロジックは標準ライブラリのみだと実装がやや複雑になるため、実装コードをモジュール化して再利用できるようにします。
