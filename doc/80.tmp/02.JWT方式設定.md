AppFlowからSalesforceへの接続において、**JWT（JSON Web Token）ベアラフロー**を使用する方法は、ユーザーインターフェースを介した手動の認可（OAuthプロンプト）を不要にし、サーバー間でセキュアに認証を完結できるため、エンタープライズ環境でのベストプラクティスとされています。

設定には「デジタル証明書」の作成と、Salesforce側での「接続アプリケーション」の高度な設定が必要です。

---

## 1. 事前準備：デジタル証明書（自己署名証明書）の作成

JWT認証では、秘密鍵で署名し、公開鍵をSalesforceに登録することで信頼関係を構築します。

1.  **ターミナル（OpenSSL）で作成する場合**:
    ```bash
    # 秘密鍵の作成
    openssl genrsa -out fldevcrm.key 2048
    # 証明書署名要求（CSR）と公開鍵の作成
    openssl req -new -key fldevcrm.key -out fldevcrm.csr
    # 自己署名証明書の作成（fldevcrm.crt）
    openssl x509 -req -sha256 -days 365 -in fldevcrm.csr -signkey fldevcrm.key -out fldevcrm.crt
    ```
2.  **Salesforce側で作成する場合**:
    * [設定] > [証明書と鍵の管理] > [自己署名証明書の作成] から作成し、公開鍵をダウンロードします。

---

## 2. Salesforce側：接続アプリケーションの設定

1.  **[アプリケーションマネージャー]** で対象の接続アプリケーションを **[編集]** します。
2.  **[デジタル署名を使用]** にチェックを入れます。
3.  手順1で作成した **`server.crt`（公開鍵）** をアップロードします。
4.  **[保存]** をクリックします。
5.  **重要：[管理] ボタンをクリックし [ポリシーを編集] を開きます。**
    * **[許可されたユーザー]**: 「管理者が承認したユーザーは事前承認済み」に変更します。
    * **[IP 制限]**: 「IP 制限を緩和」を選択します。
6.  **プロファイルの追加**:
    * [管理] 画面の下部にある **[プロファイル]** セクションで、**統合専用ユーザーのプロファイル**を追加します。これにより、JWTでの自動ログインが許可されます。



---

## 3. AWS 側：Secrets Manager への登録

AppFlow で JWT 方式を利用する場合、秘密鍵（`server.key`）などの情報を **AWS Secrets Manager** に特定の JSON 形式で保存する必要があります。

1.  **Secrets Manager** で「新しいシークレットを保存する」を選択します。
2.  **シークレットのタイプ**: 「その他のシークレット」
3.  **キー/値のペア**: 以下の形式で入力します。
    * `clientId`: Salesforceのコンシューマ鍵
    * `userId`: 統合専用ユーザーのユーザー名（例: `aws.appflow.load@example.com`）
    * `privateKey`: `server.key` の内容（`-----BEGIN RSA PRIVATE KEY-----` も含む）
    * `tokenEndpoint`: `https://login.salesforce.com/services/oauth2/token`

---

## 4. AppFlow 側：接続（コネクタプロファイル）の作成

1.  AppFlow コンソールで **[接続]** > **[Salesforce]** > **[接続を作成]**。
2.  **認証問合せの種類**: **[JSON Web Token (JWT) ベアラ]** を選択します。
3.  **シークレットを選択**: 手順3で作成した Secrets Manager のシークレットを指定します。
4.  **接続名**を入力し、**[作成]** をクリックします。

---

## 💡 JWT方式のメリットとトラブル解消

* **「Access Restricted」の回避**:
    JWT方式（かつ「管理者が承認したユーザーは事前承認済み」設定）の場合、ブラウザでのログイン画面を経由しないため、**「API 限定ユーザー」の設定をオンにしたままでも接続が可能**です。
* **エラー発生時の確認**:
    `OAUTH_APPROVAL_ERROR_GENERIC` が出る場合は、Salesforce側の接続アプリケーション設定で、**プロファイルが正しく追加されているか**、および **Client ID がシークレット内の値と一致しているか** を再確認してください。
