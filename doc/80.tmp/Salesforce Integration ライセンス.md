Salesforce Integration ライセンス（API限定の安価なライセンス）を利用している際、標準オブジェクト（取引先、連絡先、商談など）にアクセスできず、権限セットを割り当てようとしてもエラーが出るのは、**このライセンス特有の「プロファイル制限」**が原因です。

これを解決するには、手動で権限を付与するのではなく、**Salesforceがこのライセンス専用に用意している「システム権限セット」をまず割り当てる**必要があります。

---

## 解決手順：Salesforce API 統合権限セットの活用

Salesforce Integration ライセンスのユーザーには、通常の「編集」画面から標準オブジェクトの権限を付けることができません。以下の手順で専用の権限セットを有効化してください。

### 1. 専用の権限セットを割り当てる
1.  **[設定]** ➔ **[ユーザー]** ➔ **[権限セット]** を開きます。
2.  標準で用意されている **「Salesforce API Integration」** という名前の権限セットを探します。
3.  この権限セットを、AppFlow用の統合ユーザー（`dn@thinkerdn.net`）に割り当てます。
    * この権限セットは、Integration ライセンスユーザーが標準オブジェクトにアクセスするための「ベース」となります。

### 2. カスタム権限セットでオブジェクト権限を付与する
上記 1 の「ベース」を割り当てただけでは、まだ「参照・更新」などの具体的な権限が足りない場合があります。
1.  新しく **[権限セット]** を作成します。
2.  作成時の **[ライセンス]** 選択で、必ず **[Salesforce API Integration]** を選択してください（ここを「なし」にすると標準オブジェクトが出てこない場合があります）。
3.  **[オブジェクト設定]** から、AppFlowで使いたいオブジェクト（例：Account, Contact）を選択し、[参照][作成][編集] などのチェックを入れて保存します。
4.  これをユーザーに割り当てます。



---

## 注意：Integration ライセンスでアクセスできないケース
このライセンスは非常に制限が強いため、以下の点に注意してください。

* **「商談 (Opportunity)」へのアクセス**: 
    以前の仕様ではアクセスできませんでしたが、現在は専用の権限セット経由で許可されるようになっています。
* **カスタム設定/カスタムメタデータ**: 
    これらへのアクセス権も個別に権限セットで付与する必要があります。
* **共有設定 (OWD)**: 
    ライセンスに権限があっても、オブジェクトの共有設定が「非公開」で、そのユーザーにレコードが共有されていない場合はデータが見えません。統合ユーザーをロール階層の最上位に置くか、「すべて表示」権限を検討してください。

ステップ 1：権限セットの「ライセンス」設定を修正する
権限セットを新規作成する際、選択する「ライセンス」によって、設定画面に表示される項目が決まります。

[設定] ➔ [ユーザー] ➔ [権限セット] ➔ [新規] をクリック。

表示ラベル: 「AppFlow標準オブジェクトアクセス」など。

【重要】ライセンス: ここで 「なし」 ではなく、必ず 「Salesforce API Integration」 を選択してください。

※「Salesforce Integration」ではなく、「Salesforce API Integration」という名称でリストにあるはずです。

[保存] をクリック。
