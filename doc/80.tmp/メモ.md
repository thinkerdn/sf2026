Salesforce側の設定は、セキュリティの「土台」を作る非常に重要なステップです。ベストプラクティスである**「最小権限の原則」**に基づき、`Salesforce Integration` ライセンスを使用した具体的な手順を解説します。

---

## 1. プロファイルの作成（土台作り）

標準の「API専用プロファイル」をコピーして、管理しやすいカスタムプロファイルを作成します。

1.  **[設定] (⚙️) > [ユーザー] > [プロファイル]** を開きます。
2.  `Salesforce API Only System Integrations` プロファイルを探し、その横にある **[コピー]** をクリックします。
    * *※このプロファイルは「Salesforce Integration」ライセンス専用の標準プロファイルです。*
3.  **プロファイル名**: `AppFlow_Integration_Profile` と入力して **[保存]** します。
4.  **設定の確認**:
    * クローンした直後の状態で「API の有効化」と「API 限定ユーザー」にチェックが入っているはずです。
    * **重要**: このプロファイルではオブジェクト権限を最小限（すべてオフ）にしておき、データへのアクセスは次の「権限セット」で制御します。



---

## 2. 権限セットの作成（データアクセスの許可）

AppFlowが読み書きするオブジェクト（取引先など）の権限を定義します。

1.  **[設定] > [ユーザー] > [権限セット]** を開きます。
2.  **[新規]** をクリックします。
3.  **表示名**: `AppFlow_Data_Access`
4.  **ライセンス**: **[Salesforce Integration]** を必ず選択してください。
    * *※ここでライセンスを指定しないと、Integrationライセンスのユーザーに割り当てられない場合があります。*
5.  **[保存]** をクリックし、作成した権限セットの詳細画面で **[オブジェクト設定]** を選択します。
6.  連携対象のオブジェクト（例：`取引先`）をクリックし、**[編集]** を押して以下の権限を付与します：
    * **オブジェクト権限**: [参照] [作成] [編集] [すべて表示]
    * **項目権限**: AppFlowでマッピングする全項目の [参照アクセス権] と [編集アクセス権] にチェック。
    * **外部ID項目**: Upsertに使用する項目の権限を忘れずにチェックしてください。
7.  **[保存]** します。

---

## 3. 統合ユーザーの作成（実行者の登録）

実際にAppFlowがログインに使用する専用ユーザーを作成します。

1.  **[設定] > [ユーザー] > [ユーザー]** を開きます。
2.  **[新規ユーザー]** をクリックします。
3.  **ユーザー情報の入力**:
    * **姓/名**: `AppFlow` / `Integration`
    * **メールアドレス**: 管理者の受信可能なアドレス（確認メールが届きます）。
    * **ユーザー名**: `appflow.integration@yourdomain.com`（一意である必要があります）。
4.  **ライセンスとプロファイルの設定**:
    * **ユーザーライセンス**: **[Salesforce Integration]** を選択。
    * **プロファイル**: 手順1で作成した **`AppFlow_Integration_Profile`** を選択。
5.  **[保存]** します。
6.  **権限セットの割り当て**:
    * 保存後のユーザー詳細画面で、下部の **[権限セットの割り当て]** セクションの **[割り当ての編集]** をクリック。
    * 手順2で作成した **`AppFlow_Data_Access`** を右側に移動して **[保存]** します。



---

## 4. 仕上げ：パスワードの設定と接続アプリへの追加

1.  **パスワード設定**:
    * 新規ユーザー作成時に届くメールから、一度パスワードを設定してください。
    * *※JWT方式でも、ユーザーが「有効」である必要があるため、初期設定は必要です。*
2.  **接続アプリへのプロファイル追加（再確認）**:
    * 前の手順で作成した **「接続アプリケーション」** の管理画面を開き、下部の「プロファイル」セクションに、今回の **`AppFlow_Integration_Profile`** が追加されていることを確認してください。
    * これにより、「管理者が承認したユーザー（このプロファイルのユーザー）」としてJWT認証が許可されます。

Pythonを使用してSalesforceへJWT（JSON Web Token）認証で接続するには、一般的にOAuth 2.0 JWTベアラーフローを使用します。この方法は、ユーザーによる手動ログインを介さずに、サーバー間連携などの自動処理で利用されます。 [1, 2, 3, 4] 
1. 事前準備（Salesforce側の設定）
接続の前に、Salesforceで[接続アプリケーションを作成](https://help.salesforce.com/s/articleView?id=xcloud.remoteaccess_oauth_jwt_flow.htm&language=ja&type=5)する必要があります。 [5] 

* デジタル証明書の作成: 自己署名証明書（.crtファイル）と秘密鍵（.keyファイル）を生成します。
* 接続アプリの作成: [Salesforce 設定]から「アプリケーションマネージャ」を開き、新しい接続アプリを作成します。
* 「デジタル署名を使用」にチェックを入れ、生成した証明書をアップロードします。
   * OAuth スコープに api や refresh_token, offline_access などを設定します。
* ポリシーの変更: 接続アプリの「ポリシー編集」で、「許可されたユーザ」を「管理者が承認したユーザは事前承認済み」に変更し、対象のプロファイルまたは権限セットを割り当てます。 [5, 6, 7, 8, 9] 

2. Pythonの実装例
PythonでJWTを生成し、アクセストークンを取得するには PyJWT や cryptography ライブラリを使用します。 [2] 

import jwtimport requestsimport datetime
# 設定情報consumer_key = 'あなたの接続アプリのコンシューマ鍵'username = 'salesforce_user@example.com'private_key_path = 'server.key'  # 秘密鍵ファイルのパスtoken_url = 'https://login.salesforce.com' # 本番用 (Sandboxは test.salesforce.com)
# 秘密鍵の読み込みwith open(private_key_path, 'r') as f:
    private_key = f.read()
# JWTペイロードの作成payload = {
    'iss': consumer_key,
    'sub': username,
    'aud': 'https://login.salesforce.com',
    'exp': datetime.datetime.utcnow() + datetime.timedelta(minutes=3) # 有効期限
}
# JWTの署名assertion = jwt.encode(payload, private_key, algorithm='RS256')
# トークンリクエストdata = {
    'grant_type': 'urn:ietf:params:oauth:grant-type:jwt-bearer',
    'assertion': assertion
}
response = requests.post(token_url, data=data)access_token = response.json().get('access_token')instance_url = response.json().get('instance_url')

print(f"Access Token: {access_token}")

便利なライブラリ
手動でJWTを生成する代わりに、simple-salesforce ライブラリを使用すると、JWT認証を含めた一連の操作をより簡単に記述できます。

from simple_salesforce import Salesforce
sf = Salesforce(
    username='user@example.com',
    consumer_key='YOUR_CONSUMER_KEY',
    privatekey_file='server.key'
)
