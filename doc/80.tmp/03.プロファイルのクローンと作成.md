Amazon AppFlowとSalesforceを安全に接続するための、**統合専用プロファイルの作成手順**を整理しました。

ベストプラクティスとして、Salesforceが提供しているAPI連携専用のライセンス「**Salesforce Integration**」をベースとしたプロファイルの作成を推奨します。

---

## 1. クローン元プロファイルの確認と選択

Salesforceではプロファイルを一から作成することはできず、既存のプロファイルを「クローン（コピー）」して作成します。

1.  **[設定]** (右上の歯車アイコン) > **[ユーザー]** > **[プロファイル]** を開きます。
2.  一覧から、以下のプロファイルを探します。
    * **推奨**: `Salesforce API Only System Integrations`
    * **理由**: このプロファイルは「Salesforce Integration」ライセンスに紐付いており、最初から「APIのみのアクセス」に最適化されています。



---

## 2. プロファイルのクローンと作成

1.  一覧にあるプロファイル名（例：Salesforce API Only System Integrations）を直接クリックします。
2.  画面上部（または中央付近）にある [コピー] ボタンをクリックします。
2.  **プロファイル名** を入力します。
    * 例: `AppFlow_Integration_Profile`
3.  **[保存]** をクリックします。

---

## 3. プロファイルの基本設定（重要項目）

作成したプロファイルの **[編集]** 画面、または各セクションで以下の3点を設定します。

### ① システム権限の設定
* **API の有効化**: チェックが入っていることを確認します（AppFlowの接続に必須です）。
* **API 限定ユーザー**: チェックを入れます。
    * *メリット: このユーザーがブラウザ（UI）からログインできなくなるため、パスワード漏洩時のリスクを最小化できます。*

### ② パスワードポリシーの設定
1.  プロファイル詳細画面の **[パスワードポリシー]** セクションで **[編集]** をクリックします。
2.  **パスワードの有効期間**: **「無期限」** を選択します。
    * *メリット: 定期的なパスワード変更によるAppFlowの連携停止（エラー）を防止できます。*

### ③ ログイン IP 範囲の制限（セキュリティのベストプラクティス）
1.  **[ログイン IP 範囲]** セクションで **[新規]** をクリックします。
2.  AWS AppFlowが使用するIPアドレス範囲（開始IPと終了IP）を入力します。
    * *メリット: 万が一パスワードが漏洩しても、AWSのサーバー以外からのアクセスを完全に遮断できます。*



---

## 4. 注意点：オブジェクト権限について

現在のSalesforceのベストプラクティスでは、**「プロファイルではログイン制限などのシステム基盤のみを設定し、オブジェクト（取引先や商談など）へのアクセス権限は『権限セット』で付与する」**ことが推奨されています。

* **プロファイル**: 「誰が」「どこから」「いつ」ログインできるかを決める。
* **権限セット**: 「どのデータ」を「見る・書く」ことができるかを決める。
